Esta es la organización que garantizará la claridad para tu proyecto Xinfer:

1. /docs/product: Contendrá el PRD, la visión del MVP y el Roadmap.

2. /docs/architecture:
sad.md (Documento de Arquitectura).
/adr: (Architecture Decision Records) para registrar por qué eliges ciertos modelos de ML.
/diagrams: Para esquemas C4 o flujos de datos.

3. /docs/technical:
/api: Especificaciones de contratos y endpoints.
/data-schemas: Definición de los tipos de datos de entrada/salida (imprescindible para el modelo de temperatura).

4. /docs/research: Para documentos de referencia externos, artículos sobre Cell-Based Architecture o estudios de logística.
