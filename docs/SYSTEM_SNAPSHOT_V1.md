## 2. Domain Dictionary (Xinfer Cell)

Strategic objects and interfaces that define the "Physical Boundaries" of the cell.

### 🧩 Value Objects & Entities
- **ShipmentProfile (Aggregate)**: The root entity for thermal risk tracking.
- **InfluenceFactor (VO)**: Represents a single risk factor (e.g., Average Temp) with Sentiment (Mitigating/Aggravating).
- **XaiDna (VO)**: The 14-point diagnostic metadata explaining the inference result.
- **RiskScore (VO)**: Normalized probability (0.0 - 1.0) of thermal failure.
- **ShipmentIdentifier (VO)**: Multi-factor key for shipment tracking across cells.

### 🔌 Ports (Interfaces)
- **IInferenceEngine**: Strategic port for risk prediction.
- **IDataReadinessEngine**: Validates if input data is sufficient for a high-confidence inference.
- **IShipmentRepository**: persistence port for shipment records.
- **ICellEventPublisher**: Integration port for publishing events via Outbox.
- **IThermalDataSource**: Port for retrieving historical or real-time temperature data.

## 3. Tech Stack Snapshot

Current versions and technical constraints enforced in the codebase.

- **Backend**: .NET 10.0 (Native AOT ready), C# 14.
- **API**: ASP.NET Core Minimal APIs (Slim Builder), JSON Source Generation (`XinferJsonContext`).
- **Persistence**: EF Core 10, Npgsql (PostgreSQL), Outbox Pattern for eventual consistency.
- **UI**: Blazor WebAssembly, Tailwind CSS (Solid-State UI v2.0 / Bento Grid design).
- **Inference**: Target `Microsoft.ML.OnnxRuntime` (planned for Ruta B).
- **Compliance**: GxP Immutable Audit Ledger, Trace-Parent header propagation.
